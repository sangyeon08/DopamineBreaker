# 도파민 브레이커 기획서

## 1. 아이디어

- 스마트폰, 유튜브, SNS 등에 의해 집중력을 잃은 학생들에게 디지털 디톡스를 돕는 앱
- 짧고 부담 없는 디지털 휴식 루틴을 통해 자연스럽게 사용 습관 개선

## 2. 배경

- 학생을 포함한 많은 사람들이 대중교통, 이동, 식사 등 일상 속에서 무의식적으로 스마트폰 사용
- 스마트폰 과의존 실태조사 결과
  - 국내 숏폼 이용자의 23%가 이용 시간 조절에 어려움을 겪음
  - 청소년의 36.7~40%가 과의존 위험군에 해당하는 심각한 수준
- 과도한 스마트폰 및 숏폼 사용이 집중력 저하, 수면 부족 등의 문제를 유발
- 디지털 디톡스가 필요하지만 실천이 어려운 학생들을 위해 작고 실천 가능한 서비스 필요

## 3. 서비스 개요

### 3-1. 개요

도파민 브레이커는 스마트폰 과의존으로 집중력을 잃은 학생들에게 짧은 휴식을 제안해, 단순히 사용을 차단하는 방식이 아니라 잠깐 쉬어가기를 통해 자연스럽게 습관을 개선하도록 돕습니다.  
사용 기록 보여주기, 짧은 미션 주기, 미션 중 집중 모드 활성화, 성취 기록 제공 기능으로 학생 스스로 자기 관리를 할 수 있도록 설계되었습니다.

### 3-2. 주요 효과

1. **스크린타임 기록 기능**
   - 하루 스마트폰 사용 시간 및 주요 앱별 사용량을 시각화
   - 자신의 사용 패턴을 인식해 자기 관리 능력을 향상
2. **짧은 미션 제공 기능**
   - 난이도별 미션을 제공하고 스마트폰에서 벗어나 짧은 휴식을 유도
   - 즉각적인 리프레시를 통해 집중력 회복 및 건강한 습관 형성 지원
3. **미션 중 집중 모드 활성화**
   - 미션 진행 타이머 제공
   - 앱 이탈 시 “앱으로 다시 돌아와주세요”와 같은 알림으로 집중 유지 지원
4. **미션 클리어 개수 모아보기**
   - 완료 미션 수, 연속 성공 일수 등을 제공해 성취감과 동기 부여
   - 꾸준한 사용을 유도하고 자기 관리 능력 향상

## 4. 주요 기능 사용 시나리오

1. **스크린타임 기록 확인**
   - 학생이 앱을 실행해 오늘 총 사용 시간 및 앱별 사용 내역 확인
   - 예상보다 높은 사용량을 보고 다음 날 사용 시간을 줄이기로 다짐
2. **짧은 미션 수행**
   - 집중이 흐트러진 학생이 10분짜리 미션 선택
   - 스트레칭·심호흡 루틴 가이드와 함께 타이머 실행
   - 10분 후 미션 클리어 알림과 배지 제공
3. **집중 모드 활용**
   - 20분 독서 미션 진행 중 다른 앱 실행 시 “지금은 집중 모드 중입니다” 알림
   - 유튜브 대신 책을 읽고 20분을 마치며 미션 완료
4. **미션 클리어 모아보기**
   - 기록 화면에서 주간 미션 성공 횟수(예: 12개), 연속 성공 일수(예: 4일) 확인
   - 지난달 대비 성공 횟수 증가로 성취감을 느끼고 계속 도전 의지 강화

## 5. 서비스 분석

- **디지털 웰빙** (별점 3.6)
  - 디지털 기기 이용 현황을 확인하고 사용 시간을 직접 관리하도록 돕는 앱
  - _차별화_: 도파민 브레이커는 단순 제한이 아닌 짧은 미션과 보상 시스템으로 자율적 습관 형성 지원
- **Forest (집중하기)** (별점 4.6)
  - 휴대폰 미사용 시간 동안 나무가 자라는 방식으로 집중을 유도
  - _차별화_: 도파민 브레이커는 실제 행동 루틴을 실행하도록 유도해 현실 생활에서 리프레시 제공

## 8. 설치 및 실행

### 8-1. 사전 요구사항

- Node.js (v16 이상)
- Python (v3.8 이상)
- ~~MySQL (v8.0 이상)~~ → 개발 환경에서는 SQLite 자동 사용 (설치 불필요)

### 8-2. 데이터베이스 설정

**개발 환경 (SQLite - 권장):**

- 별도 설정 불필요! 앱 실행 시 자동으로 `dopamine_breaker.db` 파일이 생성됩니다.

**프로덕션 환경 (MySQL - 선택사항):**

```bash
# MySQL 접속
mysql -u root -p

# 스키마 파일 실행
source database/schema.sql

# backend/.env 파일 생성 및 설정
cd backend
cp .env.example .env
# .env 파일에서 MySQL 정보 입력
```

### 8-3. 백엔드 설정

```bash
cd backend

# 가상환경 생성
python -m venv venv

# 가상환경 활성화
# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate

# 패키지 설치
pip install -r requirements.txt

# 환경변수 설정
cp .env.example .env
# .env 파일을 열어 데이터베이스 정보 입력

# 초기 데이터 삽입
python init_data.py

# 서버 실행
python3 -m flask run --port 5001
# 백엔드 서버가 http://localhost:5001 에서 실행됩니다
```

### 8-4. 프론트엔드 설정

```bash
cd frontend

# 패키지 설치
npm install

# 환경변수 설정
cp .env.example .env

# 개발 서버 실행
npm run dev
# 프론트엔드 서버가 http://localhost:3000 에서 실행됩니다
```

### 8-5. 프로덕션 빌드

```bash
cd frontend
npm run build
```

## 9. API 엔드포인트

### 인증 (Auth)

- `POST /api/auth/register` - 회원가입
- `POST /api/auth/login` - 로그인
- `GET /api/auth/me` - 현재 사용자 정보 조회 (JWT 필요)

### 미션 (Missions)

- `GET /api/missions/presets` - AI 생성 미션 목록 조회
- `GET /api/missions/daily` - 오늘의 미션 조회
- `POST /api/missions/generate-daily` - 오늘의 미션 수동 생성
- `POST /api/missions/presets/complete` - 미션 완료
- `POST /api/missions/presets/fail` - 미션 실패/취소
- `GET /api/missions/medals` - 획득한 메달 조회 (bronze, silver, gold)
- `GET /api/missions/recent` - 최근 완료한 미션 조회

- **AI 미션 생성**: Google Gemini API를 활용하여 매일 다양한 난이도의 미션 자동 생성

- **메달 시스템**: Bronze, Silver, Gold 등급별 미션 완료 시 메달 획득
- **성취 기록**: 완료한 미션 기록 및 통계 확인
